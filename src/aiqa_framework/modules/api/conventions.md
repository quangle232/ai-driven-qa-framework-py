# API module conventions (REST · gRPC · GraphQL — no Playwright)

Pure backend testing. This module must NOT import Playwright.

## Sub-surfaces
- `rest/` — Service-Object Model over httpx. Specs call `services/*`; services call
  `ApiClient`; nothing calls httpx directly. Validate responses with pydantic
  (`models.py`), pass `schema=` + `expected_status=` so contract drift fails loudly.
  Mocks: respx + the FastAPI app via httpx ASGITransport.
- `grpc/` — typed `GameClient` is the only gRPC layer: deadline on every call,
  bearer metadata auth. Assert `grpc.StatusCode.*` on `grpc.RpcError`, not just
  payloads. Stubs generated by `uv run poe proto-gen` into `grpc/generated/`.
- `graphql/` — `GraphQLClient` (httpx POST `/graphql`). Documents live in
  `queries/`; validate the `data` payload with pydantic (`models.py`); GraphQL
  `errors` fail the call unless `allow_errors=True`. Mock via FastAPI ASGITransport.

## Rules
- One keyword layer per sub-surface (`ApiClient` / `GameClient` / `GraphQLClient`).
- Tag specs: `@tags(TAGS.API|TAGS.GRPC|TAGS.GRAPHQL, TAGS.REGRESSION, TAGS.P0/1/2)`
  + `@jira("KEY")`.
- Test data in `modules/api/testdata/` (or per sub-surface); not inline.
- Load performance testing from `modules/performance`, not here.

## Run in isolation
```bash
uv sync --extra api --extra grpc --extra graphql
uv run poe proto-gen          # gRPC stubs
uv run pytest tests/api -m "api or grpc or graphql"
```
Memory: `docs/ai/api/` (memory.md · test-case.md · testcases/).
